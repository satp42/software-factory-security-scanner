# PRD-Lite Reference Repository

## Overview

This repository contains standardized reference materials for creating Product Requirements Documents (PRD-Lites) for customer solutions using Refinery. These documents establish clear specifications for product development, ensuring alignment between client expectations and engineering implementation.

## Purpose

PRD-Lites serve as condensed yet comprehensive documents that:

- Define specific business problems and current processes
- Outline solution requirements with clear specifications
- Establish measurable success criteria
- Provide technical implementation guidelines
- Facilitate communication between stakeholders

## Repository Contents

- **8090-PRD-Lite-Template.md**: Standard structure and content requirements for Product Requirements Documents
- **rules-of-material-writing.md**: Guidelines for creating clear, precise, and effective technical documentation
- **Additional reference materials**: Supporting documents for PRD creation

## Using the Template

The PRD-Lite template follows a structured format with these key sections:

1. **Business Problem**: Defines current challenges, inefficiencies, and business impact
2. **Current Process**: Documents existing workflows, systems, and pain points
3. **Product Description**: Explains the solution and how it addresses the business problem
4. **Product Features**: Details specific functionalities and their implementation
5. **Technical Requirements**: Outlines integration needs, dependencies, and constraints
6. **Measurement**: Defines success metrics and evaluation criteria
7. **Appendix**: Contains supporting information, definitions, and references

## Documentation Standards

All PRD-Lite documents adhere to strict writing standards:

- Use clear, plain language with no ambiguity
- Present information in narrative form rather than bullet points
- Specify technical constraints and business requirements precisely
- Include explicit acceptance criteria for all features
- Maintain consistent formatting throughout the document

## Versioning

PRDs are versioned documents. Once approved, a PRD is frozen, and any changes result in a new version. This approach ensures clear tracking of requirements evolution while maintaining implementation stability.
