# Generated-prose slop patterns (formerly the de-slop-language skill)

## Core principle

Prefer plain declarative sentences. Cut throat-clearing, faux-balance, and padding. When a sentence asserts one thing, write one clause that asserts it.

Apply to all generated prose, headings, flags, blockquotes, callouts, tables, and acceptance criteria. Do not alter verbatim quoted material, proper names, or code. A blockquote, flag, callout, or review note is not quoted material: it is authored commentary and is fully in scope, even when it contains an embedded verbatim quote. Edit the commentary; preserve only the quoted string inside it.

## Rules

1. No fronted concessive phrases. Do not open a sentence with a concessive adjunct that delays the main claim.

2. No antithesis or correlative constructions that pit one clause against another for effect.

3. No meta-commentary. Do not describe what the sentence, section, or document is doing.

4. Do not stack or overload relative clauses. A single restrictive clause is fine; nested or excessive qualification is not.

5. No rule-of-three padding. Do not list three items where one or two carry the meaning.

6. No hedge stacking. Use at most one qualifier.

7. No em-dash as a dramatic pause. Use a comma, colon, parentheses, or a sentence break.

8. No escalation that negates a lesser claim to inflate a greater one.

## Quick self-check before finalizing

- Does any sentence open with a concessive adjunct that delays the main claim?

- Does any sentence pit one clause against another for effect or escalate by negation?

- Are em-dashes, semicolons, or three-item lists doing rhythmic work?

- Could any sentence lose a clause and keep its meaning?

- Does any first-person word appear anywhere outside a verbatim quote? If so, remove it (Rules 13-15).

- Did any first-person removal produce a passive verb? If so, recast it active (Rule 16).

## Discourse-level rules (beyond single sentences)

9. No formulaic question-headers. Do not phrase a section heading or its opening line as a rhetorical question the body then circles. State the subject directly. A question is fine once; it becomes a template when every section uses the same device.

10. No mirror-clause repetition. Do not repeat a clause frame for rhythm when the second copy adds no information. Say it once.

11. No restating one idea in successive sentences. If two or three sentences in a row make the same point in different words, keep the clearest one and cut the rest. A metaphor that re-asserts the prior sentence is not a new point.

12. Prefer a plain claim over a quotable aphorism. If a sentence sounds like a motto, check whether it states something testable. Replace the metaphor with the literal claim it stands in for.

## Information-density test (run last)

For each sentence, ask whether deleting the entire sentence loses any information. If it does not, the sentence is performing rather than informing. Delete it. This is the strongest check in the skill: most slop survives the sentence-level rules above but fails this one.

Run the voice check (Rules 13-16) with the same discipline as the density test. Voice violations survive every pattern rule above because they are grammatically clean; they fail only this check.

## Voice and person

13. Remove all first person. No first-person pronouns in generated content. This applies everywhere: definitions, measurements, acceptance criteria, framing sentences, section intros, summaries, and the authored text of flags and blockquotes. The only exception is a first-person word that sits inside a verbatim quoted string. There is no framing exception.

14. Name the metric, system, or actor as the subject. When a sentence defines or describes what is measured, make the metric or system the subject. When a sentence describes an action, name the responsible actor. Do not substitute a vague stand-in when a specific actor is identifiable.

15. No confessional or editorializing first person. Cut asides about the authors' candor, feelings, or intentions. State the fact without narrating the authors' relationship to it.

16. No passive voice as an escape from first person. Converting an active first-person clause into a passive one trades one defect for another and is not acceptable. Find the real subject and write the verb active. Reach for passive only when the actor is genuinely unknown or irrelevant and no active recast exists; in that case, note why. A copular definition is not passive and is the preferred form for defining a metric.

17. No prose that re-enumerates a nearby list or table. If a paragraph restates the items of a list or the column of a table that appears near it, cut the prose to a single pointer to that structure. The structured form is the source of truth.
