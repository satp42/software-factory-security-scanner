# Output File Specifications

This document describes the format and expected content for each output file generated by the meeting processor.

## Output Files Overview

| File | Purpose | Expected Length |
|------|---------|-----------------|
| `transcript.md` | Verbatim transcript with timestamps | Varies with meeting length |
| `meeting-summary.md` | Comprehensive summary | 2-4 pages for hour+ meeting |
| `decisions.md` | Explicit, implicit, deferred decisions | 1-3 pages typical |
| `ideas.md` | Feature ideas, project sparks | Highly variable |
| `action-items.md` | Tasks, next steps, follow-ups | 1-2 pages typical |
| `frameworks.md` | Mental models, philosophies | 0.5-2 pages typical |
| `status-updates.md` | Project state changes | 1-2 pages typical |
| `blockers.md` | Obstacles, risks, dependencies | 0.5-2 pages typical |

---

## 1. transcript.md

**Purpose:** Complete verbatim record of everything said in the meeting.

**Format:**
```markdown
# Meeting Transcript

[00:00:00] Speaker 1: Opening remarks and context...

[00:00:45] Speaker 2: Response to the opening...

[00:01:30] Speaker 1: [laughs] That's a good point. I think we should...

[00:02:15] [crosstalk]

[00:02:20] Speaker 2: Sorry, go ahead.

[00:02:22] Speaker 1: No, you first.
```

**Requirements:**
- Timestamps every 30-60 seconds of speaking
- Speaker identification (names or Speaker 1/2/3)
- Verbal fillers preserved (um, uh, like)
- Non-verbal cues in brackets: `[laughs]`, `[pause]`, `[crosstalk]`
- Unclear audio marked as `[inaudible]` or `[unclear]`

**Quality Check:**
- Must be COMPLETE - no summarization or skipping
- Every word matters

## 2. meeting-summary.md

**Purpose:** Comprehensive overview of the meeting for quick reference.

**Format:**
```markdown
# Meeting Summary

## Overview
- **Date/Time:** [from video context]
- **Participants:** [names/roles if identifiable]
- **Purpose:** [meeting context]

## Key Discussion Points

### [Topic 1]
- What was discussed
- Key points raised
- Outcomes/conclusions

### [Topic 2]
...

## Critical Insights
- Most important takeaway 1
- Strategic observation 2
- Notable quote: "..."

## Open Questions
- Unresolved item 1
- Topic needing follow-up 2
```

**Quality Check:**
- For hour+ meetings: expect 2-4 pages minimum
- 1-page summary for long meeting = incomplete processing

## 3. decisions.md

**Purpose:** Capture all decisions made, including implicit ones.

**Format:**
```markdown
# Decisions Made

## Explicit Decisions

### [Decision Title]
- **Timestamp:** [HH:MM:SS]
- **Decision:** [what was decided]
- **Rationale:** [why, if discussed]
- **Impact:** [what this affects]

## Implicit Decisions
- [Timestamp] Deprioritized X in favor of Y
- [Timestamp] Accepted default approach for Z

## Deferred Decisions
- **What:** [decision topic]
- **Why deferred:** [reason]
- **Revisit when:** [timeline if mentioned]

## Decision Rationale
[For major decisions, detailed reasoning captured]
```

**Key Phrases to Detect:**
- "Let's go with..."
- "We've decided..."
- "The plan is to..."
- "We're not going to..."
- "For now, let's..."

---

## 4. ideas.md

**Purpose:** Capture every idea mentioned, even casual ones.

**Format:**
```markdown
# Ideas & Project Sparks

## Feature Ideas

### [Idea Title]
- **Timestamp:** [HH:MM:SS]
- **Description:** [the idea]
- **Context:** [why it came up]
- **Related project:** [if mentioned]

## Project Sparks
- [New tool concept]
- [Standalone initiative]

## Embedded Ideas
(Ideas found in problem discussions)
- "The issue is we don't have X" → **Idea: Build X**
- "It's hard because Y" → **Idea: Solve Y**

## Raw Ideas
(Half-formed thoughts worth capturing)
- [Quick aside about...]
- [Dismissed idea that might be revisited]
```

**Quality Check:**
- Don't filter - capture everything
- Brainstorming discussion should yield many ideas
- 1-2 ideas from brainstorm = incomplete extraction

## 5. action-items.md

**Purpose:** Track all tasks and next steps.

**Format:**
```markdown
# Action Items

## Assigned Tasks

### [Task Description]
- **Timestamp:** [HH:MM:SS]
- **Owner:** [who is responsible]
- **Deadline:** [if mentioned]
- **Context:** [why this matters]

## Unassigned Tasks
- [ ] Task needing owner
- [ ] Another task without assignment

## Next Steps
1. [What happens after this meeting]
2. [Follow-up activity]

## Follow-ups
- Research: [topic to investigate]
- Contact: [person to reach out to]
- Verify: [thing to confirm]
```

**Key Phrases to Detect:**
- "I'll..." / "You'll..." / "We need to..."
- "Can you..." / "Let's make sure..."
- "Action item:"
- "Next step is..."
- "By [date]..."

## 6. frameworks.md

**Purpose:** Capture mental models and team philosophies.

**Format:**
```markdown
# Frameworks & Philosophies

## Mental Models

### [Framework Name]
- **Timestamp:** [HH:MM:SS]
- **Description:** [the mental model]
- **Application:** [how it was used in discussion]

## Principles
- "We always [principle]"
- "The rule is [guideline]"
- "Our approach is [method]"

## Philosophies
- [Team belief expressed]
- [Working principle stated]
- [Cultural value mentioned]

## Ways of Thinking
- [Novel approach to problem]
- [Interesting reframe]
- [Strategic lens applied]
```

**Key Phrases to Detect:**
- "The way I think about it..."
- "Our philosophy is..."
- Explanatory analogies
- Repeated principles

## 7. status-updates.md

**Purpose:** Track project state changes.

**Format:**
```markdown
# Status Updates

## Projects Now Live/Complete
- **[Timestamp]** [Project/Feature] is now live
  - What changed: [description]
  - Impact: [discussed effects]

## Projects In Progress
- **[Project Name]**
  - Current state: [status]
  - Progress: [what's been done]
  - Expected completion: [timeline]

## Projects Blocked/On Hold
- **[Project Name]**
  - Status: Blocked/On Hold
  - Blocker: [what's stopping it]
  - Unblock condition: [what would help]

## Changes in Priority
- [Project] moved UP because [reason]
- [Project] moved DOWN because [reason]

## Timeline Updates
- [Project] delayed to [new date]
- [Project] accelerated to [new date]
```

**Key Phrases to Detect:**
- "X is now..."
- "We shipped..."
- "That's done..."
- "We're waiting on..."
- "The new timeline is..."

---

## 8. blockers.md

**Purpose:** Identify all obstacles and risks.

**Format:**
```markdown
# Blockers & Obstacles

## Technical Blockers
- **[Timestamp]** [Blocker description]
  - Impact: [what can't proceed]
  - Proposed solution: [if discussed]

## Dependencies
- Waiting on [person] to [action]
- Need decision from [stakeholder]
- Requires [resource]

## Resource Constraints
- Time: [constraint]
- Budget: [limitation]
- Capacity: [issue]

## Knowledge Gaps
- Don't know: [unknown]
- Need to research: [topic]
- Expertise needed: [area]

## Process Blockers
- Approval needed from [entity]
- Coordination challenge with [team]

## Risks Identified
- **Risk:** [potential problem]
  - Likelihood: [if discussed]
  - Mitigation: [if discussed]
```

**Key Phrases to Detect:**
- "We can't until..."
- "We're blocked on..."
- "The issue is..."
- "We need to figure out..."
- "The risk is..."

## File Naming Convention

All output files use kebab-case with `.md` extension:
- `transcript.md`
- `meeting-summary.md`
- `decisions.md`
- `ideas.md`
- `action-items.md`
- `frameworks.md`
- `status-updates.md`
- `blockers.md`

---

## Quality Verification Checklist

Before considering processing complete:

- [ ] `transcript.md` - Full verbatim transcript (not skimmed)
- [ ] `meeting-summary.md` - Substantial summary (2+ pages for hour+ meeting)
- [ ] `decisions.md` - All explicit AND implicit decisions
- [ ] `ideas.md` - All feature ideas, even casual mentions
- [ ] `action-items.md` - All tasks and next steps
- [ ] `frameworks.md` - Mental models and philosophies
- [ ] `status-updates.md` - All project status changes
- [ ] `blockers.md` - All identified blockers

**Red Flags:**
- Summary shorter than expected for meeting length
- Few ideas from brainstorming discussion
- No status changes in status-heavy meeting
- Missing implicit content
